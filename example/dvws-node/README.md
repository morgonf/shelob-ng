# shelob-ng / DVWS-Node — worked example

[DVWS-Node](https://github.com/snoopysecurity/dvws-node) (Damn Vulnerable Web Services — Node.js)
covers 39 vulnerability classes across REST, XML, and GraphQL endpoints.

**Best for:** Injection testing (SQLi, NoSQL, OS command, XPATH, LDAP), XML injection, wide vuln coverage.

---

## Quick start

```bash
# 1. Start DVWS-Node (external)
git clone https://github.com/snoopysecurity/dvws-node.git /opt/dvws-node
cd /opt/dvws-node && docker-compose up -d
# Add to /etc/hosts: 127.0.0.1 dvws.local

# 2. Run fuzzer
cd example/dvws-node/
make setup    # fetch spec
make run-1    # basic scan
make run-2    # injection payloads
```

## Vulnerability catalogue (selected)

| Vulnerability | Category |
|---------------|----------|
| IDOR / BOLA | `/v2/users/{id}`, `/v2/notes/{id}` |
| SQLi | User and notes endpoints |
| NoSQL injection | MongoDB queries |
| XXE | XML import endpoint |
| XML bomb DoS | Large XML payload |
| OS command injection | File operations |
| SSRF | URL fetch endpoint |
| JWT secret brute-force | Auth endpoints |
| CORS misconfiguration | All endpoints |
| GraphQL introspection | `/graphql` |
| GraphQL ACL bypass | Mutation endpoints |

## Notes

- Most REST endpoints require no authentication — good for scanning without setup
- XML injection endpoints are not easily testable by JSON-focused fuzzers (manual test recommended)
- The GraphQL endpoint (`/graphql`) is alongside REST but requires a separate fuzzing approach
- Spec is auto-generated by `swagger-generator.js` — re-run it inside the container to regenerate:
  ```bash
  docker exec $(docker ps -qf name=dvws) node swagger-generator.js
  ```
